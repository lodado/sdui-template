# sdui-document-react UX 개선 플랜 (Notion 수준 이상)

> **이 문서는 AI 에이전트에게 그대로 전달하는 실행 프롬프트다.**
> 시니어 UI/UX 디자이너 관점에서 작성된 스펙이며, 각 항목은 "왜(문제) → Notion 기준 동작(스펙) → 구현 가이드(파일) → 완료 조건(AC) → 검증 방법" 순서로 되어 있다.
> **스펙에 애매한 부분이 있으면 Notion의 실제 동작을 기준으로 판단하라.**

---

## 0. 실행 규칙 (반드시 지켜라)

1. **코드를 수정하기 전에 반드시 관련 파일을 먼저 읽어라.** 이 패키지는 "하이브리드" 구조다: 블록 목록은 React가 렌더하고, 포커스된 블록 1개만 ProseMirror(`focused-block/`)가 편집한다. 이 구조를 바꾸지 마라.
2. **아키텍처를 재설계하지 마라.** 기존 패턴 위에 얹어라:
   - 블록 렌더: `src/editor/BlockNode.tsx`, `src/editor/RootBlockList.tsx`
   - 편집 로직: `src/editor/handlerLogic.ts`, `src/editor/hooks/`
   - UI 상태(메뉴/선택 등): `src/editor/uiStore.ts`
   - 블록 타입 정의: `src/block-types/<type>/` (블록 1개 = 폴더 1개)
   - 인라인 마크: `src/marks/<mark>/`
   - 스타일: `src/styles/editor.css` — CSS 변수(디자인 토큰) 사용, 하드코딩 색상 금지
3. **작업 전 감사(audit) 먼저.** 아래 각 항목은 "일부 구현돼 있을 수 있다". 예: 슬래시 메뉴(`focused-block/pm/slashMenuPlugin.ts`), 멀티 블록 선택(`hooks/useSelectionKeyboard.ts`, `rangePatchLogic.ts`), 마크다운 입력 규칙(`marks/markInputRule.ts`), 클립보드(`focused-block/pm/clipboard.ts`)는 이미 존재한다. **이미 있는 기능은 새로 만들지 말고 스펙과의 차이(gap)만 메꿔라.**
4. **Phase 하나씩 진행하라.** 한 Phase가 끝날 때마다:
   - 모노레포 루트에서 `pnpm run test` 실행 → 실패하면 고치고 다시 실행 (필수)
   - `apps/docs/src/stories/document/`에 해당 기능을 시연하는 Storybook 스토리 추가/갱신
   - 변경 요약 + AC 체크리스트 결과 보고
5. **동작이 걸린 로직에는 테스트를 남겨라.** 기존 테스트 위치: `src/**/__tests__/`. 테스트 프레임워크는 Jest. 순수 로직(예: 붙여넣기 파싱, 메뉴 필터링)은 유닛 테스트, UI 동작은 스토리로 커버.
6. **접근성은 기능의 일부다.** 모든 신규 메뉴/팝오버는: 키보드 완주 가능(Tab/화살표/Enter/Esc), `role`/`aria-*` 지정, 포커스 트랩과 복귀, `:focus-visible` 링. 이걸 빼먹으면 그 작업은 미완성이다.
7. **모션 규칙:** `transform`/`opacity`만 애니메이션. 메뉴 등장 120–150ms ease-out, 사라짐 100ms. `prefers-reduced-motion` 시 모션 제거. 토큰으로 정의해 재사용.

## 2. Phase 0 — 텍스트 편집의 "손맛" (최우선)

Notion이 좋게 느껴지는 이유의 80%는 블록 기능이 아니라 **타이핑·선택·붙여넣기의 마찰 없음**이다. 여기가 기준 미달이면 나머지는 의미 없다.

### 2.1 플레이스홀더 & 빈 상태

**문제:** 빈 문서/빈 블록에서 사용자가 뭘 해야 할지 안내가 부족하면 에디터가 "죽어있는" 느낌을 준다.

**스펙 (Notion 기준):**

- 빈 paragraph 블록에 **캐럿이 있을 때만** 흐린 텍스트 `명령어는 "/" 입력` 표시. 캐럿이 없으면 빈 블록은 아무것도 안 보임.
- heading 블록은 캐럿 유무와 무관하게 항상 `제목 1`, `제목 2`, `제목 3` 플레이스홀더 표시.
- toggle/quote/callout 빈 상태에도 각각 맞는 플레이스홀더.
- 문서 전체가 비었을 때: 첫 블록 플레이스홀더 + (viewer 모드가 아닐 때) 문서 하단 여백 클릭 시 마지막에 새 paragraph 생성.
- 플레이스홀더 색상은 토큰(예: `--doc-text-placeholder`)으로, 라이트/다크 모두 대비 확보.

**구현 가이드:** ProseMirror decoration 방식(`focused-block/pm/`)으로 캐럿-의존 플레이스홀더, 비포커스 블록은 `BlockNode.tsx`에서 CSS `:empty` + `::before` 조합. 문서 하단 클릭은 `SduiDocumentEditor.tsx`의 컨테이너 클릭 핸들러.

**AC:**

- [ ] 빈 paragraph에 포커스 → 플레이스홀더 보임 / 포커스 해제 → 사라짐
- [ ] heading은 포커스 없어도 레벨별 플레이스홀더 보임
- [ ] 문서 하단 빈 공간 클릭 → 새 블록 생성 + 캐럿 이동
- [ ] 스토리북에 "빈 문서" 스토리 존재

### 2.2 슬래시 메뉴 고도화

**문제:** 슬래시 메뉴는 블록 삽입의 관문이다. 검색 품질·키보드 동선이 나쁘면 전체 UX 평가가 떨어진다.

**스펙:**

- `/` 입력 즉시 캐럿 아래에 열림. 캐럿이 화면 하단이면 위로 플립.
- 이어서 타이핑하면 **인라인 필터링** (`/h1`, `/tod`처럼). 백스페이스로 `/`까지 지우면 닫힘.
- 필터링: 대소문자 무시 + **별칭(alias) 매칭**. 각 항목에 별칭 배열 부여. 최소:
  - heading1: `h1`, `#`, `제목`, `title` / heading2: `h2`, `##` / heading3: `h3`, `###`
  - checklist: `todo`, `할일`, `체크` / bulleted-list: `ul`, `-`, `목록` / numbered-list: `ol`, `1.`
  - code: ````", `코드`/ quote:`인용`/ callout:`콜아웃`, `강조`/ divider:`hr`, `---`, `구분선`
  - image: `img`, `사진`, `이미지` / collection: `board`, `table`, `데이터베이스`
  - 나머지 블록도 영문/한글 별칭 각 1개 이상
- 섹션 그룹핑: `기본 블록` / `미디어` / `데이터베이스` / `고급`. 검색 중에는 섹션 무시하고 관련도순 플랫 리스트.
- **최근 사용 항목** 상위 노출 (localStorage, 최대 3개, 검색어 없을 때만).
- 각 항목: 아이콘 + 이름 + 1줄 설명. 호버/키보드 하이라이트 동일 스타일.
- 키보드: ↑↓ 순환, Enter 선택, Esc 닫기(입력한 `/텍스트`는 남김), 마우스 호버와 키보드 하이라이트 충돌 없게 (마우스 이동 시에만 호버 반영).
- 매칭 결과 0개면 "결과 없음" 표시 후 계속 타이핑 시 자동 닫힘.

**구현 가이드:** `focused-block/pm/slashMenuPlugin.ts` + `editor/block-menu/BlockMenu.tsx`, `blockMenuItems.ts`. 별칭/섹션/최근사용은 `blockMenuItems.ts` 데이터 확장으로. 필터 로직은 순수 함수로 분리해 유닛 테스트.

**AC:**

- [ ] `/h1` → heading1이 첫 항목, Enter로 변환
- [ ] 한글 별칭(`/할일`) 매칭 동작
- [ ] 최근 사용 3개가 검색어 없을 때 상단 노출
- [ ] 화면 하단에서 위로 플립
- [ ] 필터 함수 유닛 테스트 (별칭, 관련도 정렬, 0건)

### 2.3 마크다운 입력 단축 & 붙여넣기

**문제:** 파워유저는 마우스를 안 쓴다. 입력 변환과 붙여넣기가 Notion급 리트머스 시험지다.

**스펙 — 입력 변환 (블록 시작에서 입력 + 스페이스):**

- `#`/`##`/`###` → heading 1/2/3, `-`·`*` → bulleted, `1.` → numbered, `[]` → checklist, `>` → quote, ` ``` ` → code, `---` → divider(스페이스 불필요, 즉시), `"` → quote
- 인라인(입력 중 자동): `**굵게**`, `*기울임*`, `` `코드` ``, `~취소선~` — `markInputRule.ts`가 있으니 커버리지 감사 후 누락만 추가
- 변환 직후 **Ctrl/Cmd+Z 한 번으로 변환만 취소**되고 입력 텍스트는 남아야 함 (Notion 동작)

**스펙 — 붙여넣기 (`focused-block/pm/clipboard.ts` 확장):**

- **여러 줄 마크다운 텍스트** 붙여넣기 → 블록 구조로 파싱 (heading/list/quote/code fence/divider 최소 지원)
- **URL 붙여넣기:**
  - 텍스트 선택 상태에서 URL 붙여넣기 → 선택 텍스트에 link 마크 적용 (블록 생성 X)
  - 빈 캐럿에 URL 붙여넣기 → 인라인 링크로 넣되, 붙여넣기 직후 작은 메뉴 표시: `링크로 유지 / 북마크로 전환 / 임베드로 전환` (Esc나 다른 입력 시 링크 유지로 확정)
- **이미지 파일/클립보드 이미지** 붙여넣기 → image 블록 생성 (업로드 핸들러는 prop으로 주입받는 구조 유지, 없으면 dataURL 프리뷰)
- 일반 텍스트 여러 줄 → 줄마다 paragraph 분할

**AC:**

- [ ] 위 입력 변환 전부 동작 + 변환 취소(Undo) 동작
- [ ] 마크다운 여러 줄 붙여넣기 → 올바른 블록 트리 (파서 유닛 테스트 필수)
- [ ] 텍스트 선택 + URL 붙여넣기 → 링크 마크
- [ ] URL 단독 붙여넣기 → 전환 메뉴 표시, 키보드로 선택 가능

### 2.4 멀티 블록 선택

**문제:** 블록 에디터의 성패는 "여러 블록을 한 덩어리로 다루기"에서 갈린다.

**스펙:**

- **진입:** 텍스트 편집 중 Esc → 현재 블록이 "블록 선택" 상태(파란 오버레이). / 블록 사이 빈 여백에서 드래그 → 러버밴드 선택. / 블록 선택 상태에서 Shift+↑↓ → 선택 확장.
- **선택 시각화:** 블록 전체에 반투명 파란 오버레이(토큰 `--doc-block-selected-bg`), 텍스트 선택과 명확히 구분.
- **선택 상태 조작:**
  - Backspace/Delete → 선택 블록 일괄 삭제
  - Ctrl/Cmd+C, X → 블록 단위 복사/잘라내기 (앱 내 재붙여넣기 시 블록 구조 보존, 외부 앱엔 마크다운 텍스트로)
  - Ctrl/Cmd+D → 복제, Enter → 선택 해제 후 마지막 블록 아래 새 블록
  - 드래그 핸들 잡고 끌면 선택된 블록 전체가 함께 이동
  - 화살표 ↑↓ → 단일 선택 이동, Esc → 선택 해제
- **부분 구현 주의:** `useSelectionKeyboard.ts`, `rangePatchLogic.ts`, `uiStore.ts`에 선택 로직이 이미 있다. 먼저 현재 지원 범위를 표로 정리해 보고한 뒤 gap만 구현하라.

**AC:**

- [ ] Esc → 블록 선택 → Shift+↓로 확장 → Backspace 일괄 삭제
- [ ] 러버밴드 드래그 선택
- [ ] 멀티 선택 복사 → 붙여넣기 시 블록 구조 보존
- [ ] 멀티 선택 드래그 이동
- [ ] 선택 오버레이가 텍스트 선택과 시각적으로 구분됨

## 3. Phase 1 — 블록 조작 폴리시 (Notion 동급)

### 3.1 드래그 핸들 & 드롭 인디케이터

**스펙:**

- 블록 호버 시 좌측 거터에 `⋮⋮`(핸들) + `+`(추가) 버튼 페이드 인 (opacity 트랜지션 100ms, 레이아웃 시프트 금지 — 거터 영역 상시 확보).
- `+` 클릭 → 아래에 빈 블록 생성 + 슬래시 메뉴 자동 오픈. Alt/Option+클릭 → 위에 생성.

- **드롭 인디케이터:** 블록 사이 수평 파란 라인(2px). **좌우 드롭 시 세로 라인** → 컬럼 생성/병합 (기존 컬럼 기능과 연결). 드래그 중인 블록은 반투명 고스트 프리뷰.
- 자동 스크롤: 드래그 중 뷰포트 상/하단 근처에서 부드럽게 스크롤.

### 3.3 키보드 내비게이션 완성

**스펙:**

- 블록 첫 위치에서 Backspace → 이전 블록과 병합(텍스트 블록끼리) 또는 리스트 해제(리스트→paragraph→병합 순).
- Tab / Shift+Tab → 리스트·토글 중첩 들여쓰기/내어쓰기.
- Ctrl/Cmd+Shift+↑↓ → 현재 블록을 위/아래로 이동.
- Ctrl/Cmd+A: 1회 → 현재 블록 텍스트 전체 선택, 2회 → 전체 블록 선택.

### 3.4 미디어 블록 (image/video/file/bookmark/embed)

**스펙:**

- **업로드 상태 머신:** 빈 상태(클릭 시 업로드 패널: 업로드/URL 임베드 탭) → 업로드 중(진행률 바 + 취소) → 완료 / 실패(에러 메시지 + 재시도 버튼). 실패를 절대 조용히 삼키지 마라.
- **이미지:** 좌우 드래그 리사이즈 핸들(호버 시 표시, 폭 % 저장), 정렬(좌/중/우), 캡션(클릭 시 하단 인라인 입력, 비어 있으면 숨김), 클릭 → 라이트박스 프리뷰(Esc 닫기).
- **북마크:** URL 메타데이터(제목/설명/파비콘/썸네일) 카드. 메타 fetch는 prop 주입 핸들러(서버 필요)로, 없으면 URL만 표시하는 폴백. 로딩 스켈레톤.
- **파일:** 아이콘 + 파일명 + 용량 + 다운로드 버튼.
- 모든 미디어 블록: 선택 시 파란 아웃라인, Backspace 삭제, Enter로 아래 새 블록.

## 4. Phase 2 — Notion 이상 (차별화)

### 4.1 접근성 (Notion보다 잘할 수 있는 지점)

- 에디터 전체 landmark/role 구조 정리, 블록 이동·삭제 등 주요 동작에 `aria-live` 안내("블록을 아래로 이동했습니다").
- 드래그 앤 드롭의 완전한 키보드 대체 수단 존재 (Ctrl+Shift+화살표 + 액션 메뉴 '이동').
- 색 대비 WCAG AA 전수 점검 (특히 placeholder, 선택 오버레이, 코드 블록).
- 스크린리더로 블록 타입 식별 가능 (`aria-label`: "제목 1", "체크리스트, 완료됨" 등).

**AC:** 축소판 a11y 체크리스트를 `packages/sdui-document-react/A11Y.md`로 작성하고 항목별 통과 여부 기록.

### 4.2 모션 & 마이크로 인터랙션 시스템

- 토큰: `--doc-duration-fast: 100ms; --doc-duration-normal: 150ms; --doc-ease: cubic-bezier(0.16,1,0.3,1)` — editor.css 상단에 정의, 전 컴포넌트 재사용.
- 메뉴/팝오버: scale(0.98)+opacity 진입. 블록 드롭: 안착 시 잔상 하이라이트 300ms 페이드.
- 체크리스트 체크: 체크박스 스프링 + 텍스트 취소선 트랜지션.
- 토글 열림/닫힘: 화살표 회전 + 콘텐츠 opacity (height 애니메이션 금지 — 레이아웃 속성).
- 전부 `prefers-reduced-motion` 가드.

### 4.3 상태 가시성

- 저장 상태 인디케이터 API: 에디터가 `onContentChange` 기반이므로, 호스트가 넘기는 `saveState: 'saved'|'saving'|'error'` prop을 받아 우상단에 미세하게 표시하는 선택적 UI 제공.
- Undo/Redo 툴바 버튼(선택적 표시) + 히스토리 스택이 비면 disabled.

### 4.4 터치/모바일 1차 대응

- 드래그 핸들: 터치에서 long-press로 드래그 시작.
- 슬래시 메뉴/액션 메뉴: 뷰포트 좁을 때 하단 시트 스타일로 전환.
- 터치 타겟 최소 44×44px (핸들, +, 체크박스).
