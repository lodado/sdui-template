# Add Block (Slash Menu + '+' Gutter Button) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Users insert new blocks via a `/` slash menu and a `+` gutter button, covering all block types (paragraph, heading 1–3, checklist, divider, callout, image, file, link) with Notion semantics (empty block converts in place, non-empty inserts below).

**Architecture:** A ProseMirror plugin in the focused-block editor detects `/` and reports open/query/close through the existing `FocusedBlockCallbacks` delegation channel. React (FocusedBlockEditor) owns menu state and renders a Radix Popover menu (same virtual-anchor pattern as `SelectionToolbar`). Selection flows to a new `SduiDocumentEditor` handler that emits existing patches (`block.setType` / `block.insert` / `block.update`) — undo/redo is free via the existing inverse-patch history.

**Tech Stack:** TypeScript, React 18, ProseMirror, `@radix-ui/react-popover` (already a dependency), Jest + Testing Library, Playwright (ssr-testing).

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-04-add-block-design.md`
- No new patch types. Only `block.insert`, `block.setType`, `block.update`, `block.delete` (existing).
- `block.update` merges shallowly; an explicit `undefined` value deletes the key (see `blockPatch.ts` `updateBlock`).
- Block types are exactly: `document.paragraph`, `document.heading` (attrs `{level: 1|2|3}`), `document.checklist`, `document.divider`, `document.callout`, `document.image`, `document.file`, `document.link`.
- React block components read: image → `attributes.src/alt`, link → `attributes.url`, file → `attributes.title` (see `block-types/image/ImageBlock.tsx`, `link/LinkBlock.tsx`).
- `NON_TEXT_BLOCK_TYPES` (SduiDocumentEditor.tsx:61) = root, divider, image, file, link — these end the inline session (block gets selected, not focused).
- Menu keywords must include Korean + English.
- Commit message format: `<type>: <description>` (feat/fix/test/docs/chore), no attribution footer.
- After ALL code changes complete: run `pnpm run test` from monorepo root; must pass (CLAUDE.md mandate).

```typescript
// packages/sdui-document/src/block-types/createDefaultBlock.ts
import type { SduiDocumentBlock } from '../blocks/schema/block'
import { createBlockId } from '../blocks/schema/ids'

const EMPTY_TEXT_STATE = () => ({ content: [], text: '' })

/**
 * Freshly-inserted block for the block menu / '+' button.
 * Per-type defaults only — callers merge menu attrs on top.
 */
export function createDefaultBlock(
  type: SduiDocumentBlock['type'],
  id: string,
  attributes?: Record<string, unknown>,
): SduiDocumentBlock {
  const blockId = createBlockId(id)

  switch (type) {
    case 'document.divider':
      return { id: blockId, type, ...(attributes ? { attributes } : {}) }

    case 'document.checklist':
      return { id: blockId, type, state: EMPTY_TEXT_STATE(), attributes: { checked: false, ...attributes } }

    case 'document.image':
    case 'document.file':
    case 'document.link':
      return { id: blockId, type, state: { text: '' }, ...(attributes ? { attributes } : {}) }

    default:
      return { id: blockId, type, state: EMPTY_TEXT_STATE(), ...(attributes ? { attributes } : {}) }
  }
}
```

```typescript
// packages/sdui-document-react/src/editor/block-menu/blockMenuItems.ts
/**
 * Block menu registry — analog of Outline app/editor/menus/block.tsx.
 * `action` decides the select flow: 'insert' patches immediately, 'file'
 * opens the file picker first, 'link' switches the menu to a URL input.
 */
export type BlockMenuItem = {
  id: string
  type: string
  title: string
  /** Plain-text glyph — no icon dependency in this package. */
  glyph: string
  keywords: readonly string[]
  attributes?: Record<string, unknown>
  action: 'insert' | 'file' | 'link'
}

export const BLOCK_MENU_ITEMS: readonly BlockMenuItem[] = [
  {
    id: 'paragraph',
    type: 'document.paragraph',

    keywords: ['text', 'paragraph', 'plain', '텍스트', '본문', '문단'],
  },
  {
    id: 'heading-1',

    title: 'Heading 1',

    action: 'insert',
    attributes: { level: 1 },
    keywords: ['heading', 'h1', 'title', '제목', '헤딩'],

  {
    id: 'heading-2',
    type: 'document.heading',
    title: 'Heading 2',
    glyph: 'H2',
    action: 'insert',
    attributes: { level: 2 },
    keywords: ['heading', 'h2', 'subtitle', '제목', '헤딩'],
  },
  {
    id: 'heading-3',
    type: 'document.heading',
    title: 'Heading 3',
    glyph: 'H3',
    action: 'insert',
    attributes: { level: 3 },

  },
  {

  {

    id: 'divider',

    title: 'Divider',
    glyph: '—',
    action: 'insert',
    keywords: ['divider', 'hr', 'rule', 'separator', '구분선', '분선'],
  },
  {
    id: 'image',
    type: 'document.image',
    title: 'Image',
    glyph: '▨',
    action: 'file',
    keywords: ['image', 'picture', 'photo', '이미지', '사진', '그림'],
  },
  {
    id: 'file',
    type: 'document.file',
    title: 'File',
    glyph: '▣',
    action: 'file',
    keywords: ['file', 'attachment', 'upload', '파일', '첨부'],
  },
  {
    id: 'link',
    type: 'document.link',
    title: 'Link',
    glyph: '⇗',
    action: 'link',
    keywords: ['link', 'url', 'bookmark', '링크', '주소'],
  },
]

export function filterBlockMenuItems(query: string): BlockMenuItem[] {
  const needle = query.trim().toLowerCase()
  if (!needle) {
    return [...BLOCK_MENU_ITEMS]
  }

  return BLOCK_MENU_ITEMS.filter(
    (item) =>
      item.title.toLowerCase().includes(needle) ||
      item.keywords.some((keyword) => keyword.toLowerCase().includes(needle)),
  )
}
```

**Interfaces:**

- Consumes: `focusedBlockSchema`, `FocusedBlockCallbacks`
- Produces (added to `FocusedBlockCallbacks` — implementers in Task 5):
  ```typescript
  /** '/' detected at block start or after whitespace — menu should open. */
  onSlashMenuOpen(anchor: { left: number; top: number; bottom: number }): void
  /** Query text after the '/' changed while the range is still valid. */
  onSlashMenuQueryChange(query: string): void
  /** Slash range destroyed (slash deleted, whitespace typed, caret left). */
  onSlashMenuClose(): void
  /** React owns menu-open state (also true for '+'-opened menus). */
  isSlashMenuOpen(): boolean
  /** Arrow/Enter/Escape routed to the menu while open. true = consumed. */
  onSlashMenuKey(key: 'up' | 'down' | 'enter' | 'escape'): boolean
  ```
- Produces (plugin module):
  - `buildSlashMenuPlugin(callbacks: FocusedBlockCallbacks): Plugin`
  - `getSlashRange(state: EditorState): { from: number; to: number } | null` — `/`+query span; Task 5 deletes it on select.

Design notes for the implementer:

- Plugin state: `{ slashPos: number | null }` under a `PluginKey('slashMenu')`.
- In `state.apply(tr, value)`: map `slashPos` through `tr.mapping`; re-validate (doc char at `slashPos` is `/`, caret ≥ `slashPos + 1`, text between contains no whitespace, char before `slashPos` is doc start or whitespace). Invalid → `null`. If currently `null` and `tr.docChanged`, test the text before the caret against `/(?:^|\s)\/([^\s/]*)$/` — on match set `slashPos` to the matched `/` position.
- Plugin `view.update(view, prevState)`: compare previous/current `slashPos`; skip all callbacks while `view.composing` (IME). `null → pos`: `callbacks.onSlashMenuOpen(view.coordsAtPos(pos))`. Open and query changed: `onSlashMenuQueryChange(doc.textBetween(slashPos + 1, selection.from))`. `pos → null`: `onSlashMenuClose()`.
- `props.handleKeyDown`: when `callbacks.isSlashMenuOpen()` returns true, map ArrowUp/ArrowDown/Enter/Escape to `onSlashMenuKey('up'|'down'|'enter'|'escape')` and return its result. Registered BEFORE `buildFocusedBlockKeymap` in `editorState.ts` so Enter/Escape never reach split/escape delegation while the menu is open.
- The single-textblock schema means positions are plain text offsets +1 (doc → paragraph → text); use `state.doc.textBetween(1, pos)` style helpers, verify offsets in tests.

```typescript
// packages/sdui-document-react/src/focused-block/__tests__/slashMenuPlugin.test.ts
import { TextSelection } from 'prosemirror-state'
import { EditorState } from 'prosemirror-state'

import type { FocusedBlockCallbacks } from '../pm/keymapDelegation'
import { buildSlashMenuPlugin, getSlashRange } from '../pm/slashMenuPlugin'
import { focusedBlockSchema } from '../pm/schema'

const noopCallbacks: FocusedBlockCallbacks = {
  onSplit: () => {},
  onMergeBackward: () => {},
  onIndent: () => {},
  onOutdent: () => {},
  onNavigate: () => {},
  onTurnInto: () => {},
  onEscape: () => {},
  onMoveBlock: () => {},
  onBlockAction: () => {},
  onSlashMenuOpen: () => {},
  onSlashMenuQueryChange: () => {},
  onSlashMenuClose: () => {},
  isSlashMenuOpen: () => false,
  onSlashMenuKey: () => false,
}

function stateWithPlugin(): EditorState {
  return EditorState.create({
    schema: focusedBlockSchema,
    plugins: [buildSlashMenuPlugin(noopCallbacks)],
  })
}

function typeText(state: EditorState, text: string): EditorState {
  return state.apply(state.tr.insertText(text, state.selection.from))
}

describe('slashMenuPlugin', () => {
  test('slash at block start opens a range', () => {
    const state = typeText(stateWithPlugin(), '/')
    expect(getSlashRange(state)).toEqual({ from: 1, to: 2 })
  })

  test('query grows with typed characters', () => {
    const state = typeText(typeText(stateWithPlugin(), '/'), 'head')
    expect(getSlashRange(state)).toEqual({ from: 1, to: 6 })
  })

  test('slash mid-word does not open', () => {
    const state = typeText(typeText(stateWithPlugin(), 'a'), '/')
    expect(getSlashRange(state)).toBeNull()
  })

  test('slash after whitespace opens', () => {
    const state = typeText(typeText(stateWithPlugin(), 'a '), '/')
    expect(getSlashRange(state)).not.toBeNull()
  })

  test('whitespace in query closes the range', () => {
    const state = typeText(typeText(stateWithPlugin(), '/he'), ' ')
    expect(getSlashRange(state)).toBeNull()
  })

  test('deleting the slash closes the range', () => {
    let state = typeText(stateWithPlugin(), '/')
    state = state.apply(state.tr.delete(1, 2))
    expect(getSlashRange(state)).toBeNull()
  })

  test('moving the caret before the slash closes the range', () => {
    let state = typeText(typeText(stateWithPlugin(), '/'), 'he')
    state = state.apply(state.tr.setSelection(TextSelection.create(state.doc, 1)))
    expect(getSlashRange(state)).toBeNull()
  })
})
```

Implementation notes:

- `view === 'menu'`: `<div role="listbox" aria-label="Insert block">` with buttons `role="option"` `aria-selected={index === activeIndex}` `data-active={index === activeIndex || undefined}`; `onMouseDown={(e) => e.preventDefault()}` (keep PM selection alive, same policy as toolbar buttons) and `onClick={() => onSelect(item)}`. Empty `items` renders a "No results" row.
- `view === 'link'`: `<form>` with a URL `<input autoFocus placeholder="https://…">`, submit calls `onSubmitLink(value)`, Escape in the input calls `onClose()`.
- CSS: `.sdui-doc-block-menu` panel + `[data-active]` highlight, follow existing `.sdui-doc-toolbar` styles in `editor.css`.

```typescript
// Notion semantics: an empty block converts in place, a non-empty block
// gets a new sibling below. Returns the id that received the type.
const applyMenuType = (
  blockId: string,
  item: BlockMenuItem,
  attributes: Record<string, unknown> | undefined,
  extraState?: Record<string, unknown>,
): string | null => {
  const source = findBlockById(docRef.current, blockId)
  const isEmpty = getInlineContentLength(blockInlineContent(source)) === 0

  if (isEmpty) {
    const patches: SduiDocumentPatch[] = [
      {
        type: 'block.setType',
        blockId: createBlockId(blockId),
        blockType: item.type,
        ...(attributes ? { attributes } : {}),
      },
    ]
    if (extraState) {
      patches.push({ type: 'block.update', blockId: createBlockId(blockId), state: extraState })
    }
    latest.current.applyPatches(patches)
    return blockId
  }

  const flattened = flattenDocumentBlocks(docRef.current)
  const location = flattened.find((candidate) => candidate.id === blockId)
  if (!location?.parentId) {
    return null
  }

  const newId = latest.current.generateBlockId()
  const block = createDefaultBlock(item.type, newId, attributes)
  latest.current.applyPatches([
    {
      type: 'block.insert',
      parentId: createBlockId(location.parentId),
      index: location.index + 1,
      block: extraState ? { ...block, state: { ...block.state, ...extraState } } : block,
    },
  ])
  return newId
}
```
